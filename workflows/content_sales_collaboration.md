# 콘텐츠-세일즈 협업 워크플로우

## 개요

세일즈팀(CM)이 현장에서 체득한 인사이트를 콘텐츠팀과 공유하고,
콘텐츠팀이 이를 바탕으로 세일즈 마케팅 퍼널용 콘텐츠를 제작하는 협업 구조.

## 핵심 원칙

```
"현장에서 체득한 정보 → 세일즈 마케팅 퍼널용 콘텐츠"
```

- 세일즈팀: **수요 발굴** (잠재 고객이 뭘 원하는지)
- 콘텐츠팀: **수요 충족** (원하는 콘텐츠 제작)
- **양방향 피드백**: 콘텐츠 성과 → 세일즈팀 공유 → 개선

---

## 협업 흐름

```
[CM/세일즈팀]                           [콘텐츠팀]
     │                                      │
     │  ① 현장 인사이트 공유                 │
     │  "인니 물류업체들이 cross-border       │
     │   fulfillment 데이터에 관심 많음"      │
     ├─────────────────────────────────────→│
     │                                      │
     │                            ② 인사이트 분석
     │                               콘텐츠 주제 기획
     │                                      │
     │                            ③ 콘텐츠 제작
     │                               (조사원 → 팩트체커 → 구조화 → 에디터)
     │                                      │
     │  ④ 세일즈 퍼널용 콘텐츠 제공           │
     │←─────────────────────────────────────┤
     │  - LinkedIn 글                        │
     │  - Lead Magnet PDF                   │
     │  - 이메일 Hook용 데이터               │
     │                                      │
     │  ⑤ 콘텐츠 활용                        │
     │  - 리드에게 공유                       │
     │  - 이메일/LinkedIn 메시지에 첨부        │
     │                                      │
     │  ⑥ 성과 피드백                        │
     │  "오픈율 45%, 미팅 전환 2건"           │
     ├─────────────────────────────────────→│
     │                                      │
     │                            ⑦ 콘텐츠 개선
     │                               후속 콘텐츠 기획
     │                                      │
```

## 인사이트 공유 항목

### CM이 공유해야 할 정보

| 항목 | 예시 | 활용 |
|-----|------|------|
| **잠재 고객 관심 주제** | "일본 리테일러들이 SEA 진출 데이터 필요" | 콘텐츠 주제 기획 |
| **자주 받는 질문** | "마켓플레이스별 수수료 비교 있나요?" | FAQ 콘텐츠, Lead Magnet |
| **반응 좋았던 메시지** | "Shopee vs Lazada 비교 데이터 언급 시 응답률 높음" | 콘텐츠 Hook 개선 |
| **경쟁사 언급** | "Similar Web 쓴다고 했는데 SEA 데이터 부족" | 차별점 강조 콘텐츠 |
| **구매 장벽** | "가격보다 데이터 정확도가 관건" | 신뢰 구축 콘텐츠 |
| **산업 트렌드** | "물류업체들의 크로스보더 관심 급증" | 트렌드 분석 콘텐츠 |

### 공유 형식

```yaml
파일: workspace/messages/insight_YYYYMMDD_[country].md

내용:
  - 시장: [국가명]
  - 기간: [YYYY-MM-DD ~ YYYY-MM-DD]
  - 접촉 건수: [N건]

인사이트:
  1. 관심 주제:
     - [주제1]: [빈도/중요도]
     - [주제2]: [빈도/중요도]

  2. 자주 받는 질문:
     - "[질문1]"
     - "[질문2]"

  3. 반응 좋았던 표현:
     - "[표현1]" - 응답률 [N]%

  4. 경쟁사 언급:
     - [경쟁사1]: [불만족 포인트]

  5. 구매 장벽:
     - [장벽1]

콘텐츠 요청:
  - [구체적 콘텐츠 요청사항]
```

## 콘텐츠 유형별 협업

### 1. 시장 리포트 (Lead Magnet)

```yaml
트리거: CM이 "시장 현황 데이터" 질문을 자주 받음

프로세스:
  CM → content_director: "인도네시아 이커머스 현황 리포트 필요"
  content_director → researcher_region1: 조사 지시
  researcher_region1 → fact_checker: 검증
  fact_checker → content_structurer: 구조화
  content_structurer → content_editor: 작성
  content_editor → CM: 최종 PDF 전달

산출물:
  - outputs/reports/market_report_YYYYMMDD_indonesia.pdf
  - 요약 LinkedIn 글
```

### 2. 비교 분석 콘텐츠

```yaml
트리거: CM이 "A vs B 비교" 질문을 자주 받음

프로세스:
  CM → content_director: "Shopee vs Lazada 비교 콘텐츠 필요"
  content_director → 조사원 + 팩트체커: 데이터 수집/검증
  content_editor: 비교 인포그래픽 + 글 작성
  CM: 검수 후 활용

산출물:
  - LinkedIn 글 (비교 하이라이트)
  - 비교표 이미지
  - 상세 비교 PDF
```

### 3. 트렌드 분석 콘텐츠

```yaml
트리거: CM이 "최근 트렌드" 관심 파악

프로세스:
  CM → content_director: "베트남 D2C 브랜드 트렌드 콘텐츠 필요"
  조사원: 최신 뉴스/데이터 수집
  content_editor: 트렌드 분석 글 작성
  CM: 검수 후 활용

산출물:
  - LinkedIn 스레드
  - X 스레드
  - 이메일 Hook용 데이터 포인트
```

### 4. 이메일 Hook용 데이터

```yaml
트리거: CM이 콜드 이메일 오픈율 개선 필요

프로세스:
  CM → content_director: "일본 시장용 Hook 데이터 필요"
  조사원: 최신 수치/팩트 수집
  content_editor: Hook 문장 작성
  CM: 메시지에 활용

산출물:
  - 데이터 포인트 3-5개
  - Hook 문장 예시
```

## 콘텐츠팀 협업 의무

### content_director

```yaml
의무:
  - CM/세일즈팀 콘텐츠 요청 시 **48시간 내 응답**
  - 인사이트 기반 콘텐츠 주제 기획
  - 세일즈 퍼널 단계별 콘텐츠 전략 수립
  - CM별 시장 특화 콘텐츠 우선순위 조정

주간 체크:
  - [ ] CM 인사이트 파일 리뷰 (workspace/messages/insight_*.md)
  - [ ] 콘텐츠 요청 처리 현황 확인
  - [ ] 콘텐츠 성과 피드백 수집
```

### content_editor

```yaml
의무:
  - CM 검수 요청 시 **24시간 내 응답**
  - 시장별 톤/표현 조정
  - 세일즈용 숏폼 콘텐츠 제작 (Hook, 요약)
  - 비주얼 에이드 (비교표, 인포그래픽)

산출물 종류:
  - 롱폼: LinkedIn 글, Substack, 리포트
  - 숏폼: X 스레드, 이메일 Hook, 데이터 포인트
  - 비주얼: 비교표, 차트, 인포그래픽
```

### researcher_region1/2/3

```yaml
의무:
  - CM 요청 시장 **우선 조사**
  - 현장 인사이트 기반 데이터 수집
  - 리드 대상 기업 정보 조사 지원

우선순위 조정:
  - 일반 콘텐츠 조사 < CM 요청 조사
  - CM 긴급 요청 시 즉시 대응
```

## 협업 채널

### 인사이트 공유 (CM → 콘텐츠팀)

```yaml
방법: 파일 기반
위치: workspace/messages/insight_YYYYMMDD_[country].md
주기: 주 1회 이상

알림:
  - 파일 생성 시 content_director에게 알림
  - 긴급 요청 시 직접 언급
```

### 콘텐츠 요청 (CM → 콘텐츠팀)

```yaml
방법: 인사이트 파일 내 "콘텐츠 요청" 섹션

포함 사항:
  - 요청 콘텐츠 유형 (리포트/비교/트렌드)
  - 타깃 국가/시장
  - 활용 목적 (Lead Magnet/이메일 Hook/LinkedIn)
  - 희망 기한
```

### 콘텐츠 전달 (콘텐츠팀 → CM)

```yaml
방법: 파일 기반
위치:
  - workspace/drafts/ (초안)
  - outputs/ (최종)

알림:
  - 완료 시 해당 CM에게 알림
  - 검수 요청 포함
```

### 성과 피드백 (CM → 콘텐츠팀)

```yaml
방법: 인사이트 파일 내 "성과 피드백" 섹션

포함 사항:
  - 활용 콘텐츠
  - 오픈율/응답률
  - 미팅 전환
  - 개선 요청사항
```

|-----------|------|------|

### Interest (관심)

| 콘텐츠 유형 | 목적 | 담당 |
|-----------|------|------|

| 케이스 스터디 | 신뢰 구축 | content_editor |

### Decision (결정)

| 콘텐츠 유형 | 목적 | 담당 |
|-----------|------|------|
| 데모 가이드 | 전환 유도 | content_editor |
| ROI 계산기 | 가치 정량화 | content_editor |

## 성과 측정

### 콘텐츠 성과 지표

| 지표 | 측정 방법 | 목표 |
|-----|----------|------|
| 콘텐츠 활용률 | CM 활용 콘텐츠 수 / 제작 콘텐츠 수 | 80%+ |
| 오픈율 기여 | 콘텐츠 포함 이메일 오픈율 | 30%+ |
| 미팅 전환 기여 | 콘텐츠 언급 미팅 수 | 주 2건+ |
| CM 만족도 | 콘텐츠 검수 통과율 | 90%+ |

### 협업 효율 지표

| 지표 | 측정 방법 | 목표 |
|-----|----------|------|
| 요청 응답 시간 | 콘텐츠 요청 → 초안 완료 | 48시간 내 |
| 인사이트 반영률 | 인사이트 기반 콘텐츠 수 / 전체 | 50%+ |
| 피드백 반영률 | 피드백 반영 콘텐츠 수 | 100% |

## 주간 협업 사이클

```
월요일: CM 인사이트 파일 업데이트
화요일: content_director 인사이트 리뷰 + 콘텐츠 주제 기획
수요일: 조사 + 콘텐츠 제작
목요일: CM 검수 + 수정
금요일: 콘텐츠 전달 + 성과 피드백 수집
```
